# 05 - Configurando o VS Code

O VS Code será nosso editor principal durante os estudos.

Ele foi escolhido porque é leve, muito usado no mercado, tem boa integração com JavaScript, Git, GitHub, terminal e extensões.

## O que é o VS Code

VS Code é um editor de código.

É nele que vamos abrir o projeto, ler arquivos, escrever código, usar o terminal e acompanhar os exercícios.

## Primeiros passos

Antes de começar a programar, confira se você consegue:

1. abrir o VS Code;
2. abrir a pasta do projeto;
3. navegar pelas pastas e arquivos;
4. abrir arquivos `.md`;
5. abrir arquivos `.js`;
6. usar o terminal integrado;
7. visualizar arquivos Markdown.

## Como abrir o projeto

No VS Code:

1. clique em `File`;
2. clique em `Open Folder`;
3. escolha a pasta do projeto;
4. confirme a abertura.

Depois disso, os arquivos devem aparecer na lateral esquerda do editor.

## Terminal integrado

O terminal integrado fica dentro do VS Code.

Para abrir:

```txt
Terminal > New Terminal
```

Ou use o atalho:

```txt
Ctrl + '
```

Se o terminal abrir na parte de baixo do editor, está tudo certo.

## Extensões recomendadas

Estas extensões ajudam no estudo:

| Extensão | Para que serve |
| --- | --- |
| Code Runner | Executar códigos simples com mais facilidade. |
| Error Lens | Mostrar erros e avisos de forma mais visível. |
| Git Graph | Visualizar o histórico do Git de forma gráfica. |
| Auto Rename Tag | Renomear tags HTML automaticamente. |
| Live Server | Abrir páginas HTML no navegador com atualização automática. |
| Material Icon Theme | Melhorar a visualização dos ícones dos arquivos. |
| Markdown Preview Github Styling | Visualizar Markdown com aparência parecida com GitHub. |

## O que é obrigatório agora

No primeiro momento, o mais importante é:

- conseguir abrir o projeto;
- conseguir ler os materiais;
- conseguir criar ou editar arquivos;
- conseguir abrir o terminal;
- entender onde ficam os exercícios.

As extensões ajudam, mas não substituem prática.

## Dica importante

Não tente configurar tudo de uma vez.

Primeiro deixe o básico funcionando. Depois, conforme as aulas avançarem, vamos melhorando o ambiente.
