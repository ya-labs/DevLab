# 04 - GitHub na prática

Durante os estudos, vamos usar GitHub para organizar o código e acompanhar a evolução de cada pessoa.

No começo, você não precisa dominar Git nem GitHub. Mas precisa entender o papel deles no nosso fluxo.

## O que é Git

Git é uma ferramenta de controle de versão.

Na prática, ele ajuda a registrar o histórico das alterações feitas em um projeto.

Com Git, conseguimos saber:

- quais arquivos foram alterados;
- quando uma alteração foi feita;
- quem fez a alteração;
- qual era o estado anterior do código;
- como voltar ou comparar versões.

## O que é GitHub

GitHub é uma plataforma onde podemos guardar repositórios Git na internet.

Pense nele como um lugar para hospedar projetos, colaborar com outras pessoas e mostrar sua evolução como dev.

## Conceitos importantes

| Conceito | Explicação simples |
| --- | --- |
| Repositório | Pasta de um projeto controlada pelo Git. |
| Fork | Uma cópia de um repositório para sua conta. |
| Clone | Baixar o repositório para seu computador. |
| Commit | Registro de uma alteração feita no projeto. |
| Push | Enviar commits do seu computador para o GitHub. |
| Pull Request | Pedido para revisar ou juntar alterações em outro repositório. |

## Como vamos usar na mentoria

O fluxo inicial será:

1. Você acessa o repositório do DevLab.
2. Cria um fork para sua conta.
3. Clona o seu fork no computador.
4. Faz os exercícios no seu próprio fork.
5. Envia o link da entrega no Discord quando for solicitado.

## Por que isso é importante

GitHub não serve apenas para guardar código.

Ele também mostra:

- organização;
- constância;
- evolução;
- histórico de estudo;
- capacidade de trabalhar com ferramentas reais;
- cuidado com o próprio código.

## O que você deve entender nesta semana

Nesta primeira semana, o objetivo é apenas entender a ideia geral.

Você não precisa decorar todos os comandos agora.

O mais importante é saber que o GitHub será usado para acompanhar sua jornada e organizar suas entregas.
