# 03 - Por que vamos começar com JavaScript

JavaScript será nossa primeira linguagem porque ela combina muito bem com o objetivo da turma: aprender lógica de programação e entrar no desenvolvimento front-end.

Ela é uma linguagem muito usada no mercado e aparece em vários contextos:

- sites;
- sistemas web;
- aplicações front-end;
- APIs;
- automações;
- projetos com React;
- aplicações com Node.js.

Pesquisa sobre linguagens, 2024: [link](https://pesquisa.codigofonte.com.br/2024)

## O que é JavaScript

JavaScript é uma linguagem de programação.

Com ela, conseguimos criar instruções para o computador executar.

Exemplo simples:

```js
const nome = "Nícolas";

console.log("Olá, " + nome);
```

Esse código guarda um nome e depois mostra uma mensagem.

No começo, isso parece simples demais. Mas a base da programação nasce exatamente daí:

- guardar informações;
- processar dados;
- tomar decisões;
- repetir ações;
- organizar código;
- resolver problemas.

## Por que não começar direto com React?

React é uma ferramenta muito usada no front-end moderno, mas ele depende de fundamentos.

Antes de estudar React, você precisa entender bem:

- variáveis;
- funções;
- objetos;
- arrays;
- condicionais;
- loops;
- eventos;
- módulos;
- chamadas HTTP;
- organização de código.

Se você pula essa base, React vira apenas cópia de código.

Se você constrói essa base, React passa a fazer sentido.

## O que vamos aprender primeiro

No começo da trilha, vamos focar em:

1. como um programa funciona;
2. o que são entrada, processamento e saída;
3. o que é um algoritmo;
4. como resolver exercícios;
5. como encontrar e entender erros;
6. primeiros códigos em JavaScript.

## JavaScript é só para navegador?

Não.

JavaScript nasceu muito ligado ao navegador, mas hoje também pode rodar fora dele usando Node.js.

Mesmo assim, como nosso foco inicial é front-end, vamos usar JavaScript pensando principalmente em páginas web e aplicações modernas.

## Ideia principal

JavaScript não foi escolhido por ser a única linguagem boa.

Ele foi escolhido porque é prático, muito usado no mercado e permite evoluir de forma natural para HTML, CSS, DOM, APIs, React e projetos completos.
