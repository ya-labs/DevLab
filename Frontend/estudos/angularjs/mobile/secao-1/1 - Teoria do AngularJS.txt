O que é o AngularJS

É um framework JavaScript mantido pelo Google que facilita criar aplicações 
web dinâmicas e interativas. Ele transforma [HTML estático]1 em [HTML dinâmico]2 
que responde a ações do usuário.

[1] - HTML ESTÁTICO [ 
    O conteúdo nunca muda sem recarregar a página.
    <!-- Sempre mostra "João" -->
    <h1>Olá, João!</h1>
    <p>Bem-vindo ao site</p>

    Características:
    Mesmo conteúdo para todos os usuários
    Se você quer mudar algo, precisa editar o arquivo .html e recarregar
    Sem interatividade
    Exemplo: Blog antigo, documentação estática
]
[2] - HTML DINÂMICO [
    O conteúdo muda sem recarregar a página, baseado em ações do usuário.
    <!-- AngularJS substitui {{nome}} automaticamente -->
    <h1>Olá, {{nome}}!</h1>
    <input ng-model="nome" placeholder="Digite seu nome">

    Características:
    Conteúdo muda conforme você interage
    Sem recarregar a página
    Diferente para cada usuário
    Exemplo: Gmail, Trello, redes sociais
]

Em poucas palavras: uma ferramente que deixa seu site interativo, sem precisa recarregar a página inteira.

Como funciona:
Características principais {
    1 - Two-way Data Binding (Ligação em duas vias) {
        Quando você muda algo no HTML, JavaScript muda automaticamente (e vice-versa)

        Exemplo: Campo de entrada que atualiza o título em tempo real

        HTML:
            <div ng-app="minhaApp" ng-controller="MeuController">
                <input ng-model="nome" placeholder="Digite seu nome">
                <h1>Olá, {{nome}}!</h1>
            </div>

        JavaScript:
            var app = angular.module('minhaApp', []);
            app.controller('MeuController', function($scope) {
                $scope.nome = '';
            });

        Como funciona:
            • Usuario digita no input
            • ng-model vincula o valor do input à variável $scope.nome
            • {{nome}} no HTML é atualizado automaticamente
            • Se você mudar $scope.nome via JavaScript, o input também atualiza
    }

    2 - Templates (modelos) {
        Você escreve {{ variavel }} no HTML e AngularJS substitui pelos dados reais.

        Exemplo: Mostrar dados dinâmicos no HTML

        HTML:
            <div ng-app="minhaApp" ng-controller="MeuController">
                <h1>{{titulo}}</h1>
                <p>Mensagem: {{mensagem}}</p>
                <p>Quantidade: {{quantidade}}</p>
            </div>

        JavaScript:
            var app = angular.module('minhaApp', []);
            app.controller('MeuController', function($scope) {
                $scope.titulo = 'Bem-vindo ao AngularJS';
                $scope.mensagem = 'Isso é um template dinâmico';
                $scope.quantidade = 42;
            });

        Resultado na tela:
            Bem-vindo ao AngularJS
            Mensagem: Isso é um template dinâmico
            Quantidade: 42
    }

    3 - Controllers {
        Código JavaScript que controla a lógica da página.

        Exemplo: Lógica para calcular total de compras

        HTML:
            <div ng-app="minhaApp" ng-controller="CarrinhoController">
                <h2>Carrinho de Compras</h2>
                <p>Produto: {{produto}}</p> // Mostra o nome do produto ("Livro de AngularJS" que vem do js)
                <p>Preço: R$ {{preco}}</p> // Mostra o preço do produto ("50" que vem do js)
                <input ng-model="quantidade" type="number"> // input pra digitar a quantidade (por padrão vem como "1" do js)
                <button ng-click="calcularTotal()">Calcular Total</button> // Botão que chama a função calcularTotal() ao ser clicado
                <p>Total: R$ {{total}}</p> // Mostra o total (js faz o calculo e trás o resultado aqui)
            </div>

        JavaScript:
            var app = angular.module('minhaApp', []);
            app.controller('CarrinhoController', function($scope) {
                $scope.produto = 'Livro de AngularJS'; //define o valor das variáveis
                $scope.preco = 50;
                $scope.quantidade = 1;
                $scope.total = 0;
                
                $scope.calcularTotal = function() { //função que calcula o total
                    $scope.total = $scope.preco * $scope.quantidade; //preço * quantidade
                };
            });
    }

    4 - Services {
        Funções reutilizáveis que vários controllers usam.

        Exemplo: Serviço reutilizável para cálculos

        HTML:
            <div ng-app="minhaApp" ng-controller="VendedorController">
                <p>Total vendido: R$ {{totalVendido}}</p> //Mostra o totalVendido que vem do js
                <button ng-click="adicionarVenda(100)">Adicionar R$ 100</button> //botão que ao ser clicado chama a função de adicionarVenda, passando R$ 100
                <button ng-click="adicionarVenda(50)">Adicionar R$ 50</button> // e aqui 50
            </div>

        JavaScript:
            var app = angular.module('minhaApp', []);
            
            // Criar um Service
            app.service('VendasService', function() {
                var total = 0; //total como var sem this. = só pode ser usado dentro da função
                
                this.adicionarVenda = function(valor) { //função de adicionarVenda com this. = pode ser chamada e usada no controller
                    total += valor; //a função recebe um valor (50, por exemplo) e adiciona ao total, retornando ele
                    return total;
                };
                
                this.obterTotal = function() { //função de obterTotal com this. = pode ser chamada e usada no controller
                    return total; //só retorna o total
                };
            });
            
            // Usar o Service no Controller
            app.controller('VendedorController', function($scope, VendasService) {
                $scope.totalVendido = 0;
                
                $scope.adicionarVenda = function(valor) {
                    $scope.totalVendido = VendasService.adicionarVenda(valor);
                };
            });

        Vantagem: Você pode reutilizar VendasService em vários controllers
    }

    5 - Directives {
        Instruções especiais no HTML que AngularJS entende.

        Exemplo: Usando directives para controlar o HTML

        HTML:
            <div ng-app="minhaApp" ng-controller="ListaController">
                <!-- ng-if: mostra/esconde conforme condição -->
                <p ng-if="logado">Bem-vindo, usuário!</p>
                
                <!-- ng-show/ng-hide: mostra ou esconde com CSS -->
                <p ng-show="mostrarMensagem">Essa mensagem está visível</p>
                
                <!-- ng-repeat: repete elemento para cada item da lista -->
                <ul>
                    <li ng-repeat="item in lista">{{item}}</li>
                </ul>
                
                <!-- ng-class: aplica classe CSS dinamicamente -->
                <p ng-class="{ativo: isAtivo}">Texto com classe dinâmica</p>
                
                <!-- ng-disabled: desabilita elemento conforme condição -->
                <button ng-disabled="!logado">Enviar</button>
            </div>

        JavaScript:
            var app = angular.module('minhaApp', []);
            app.controller('ListaController', function($scope) {
                $scope.logado = true;
                $scope.mostrarMensagem = true;
                $scope.isAtivo = true;
                $scope.lista = ['Maçã', 'Banana', 'Laranja'];
            });

        ═══════════════════════════════════════════════════════════════
    }

Arquitetura básica {
    O modelo MVC no AngularJS
    [HTML + Templates]
            ↓
    [Controller]  ← Lógica
            ↓
    [Services/Data] ← Dados reutilizáveis

    Separação clara:
    - View (HTML): o que o usuário vê
    - Controller: lógica que controla o que acontece
    - Model/Services: dados e funções reutilizáveis
}

AngularJS vs JavaScript puro {
    ASPECTO                    | JAVASCRIPT PURO                        | ANGULARJS
    ────────────────────────────────────────────────────────────────────────────────────
    Atualizar página           | Manual (document.getElementById)       | Automático (Two-way binding)
    Validação                  | Você escreve tudo                      | Built-in (pronto para usar)
    Reutilização de código     | Difícil e confusa                      | Fácil com Services
    Organização                | Pode ficar bagunçada                   | Estruturada em MVC
    Tempo de desenvolvimento   | Mais lento                             | Muito mais rápido
}

AngularJS + Cordova {
    Por que usar os dois?

    Cordova -> transforma app web em app mobile (Android/iOS)
    AngularJS -> Deixa a app dinâmica e responsiva
    
    Fluxo de funcionamento final:
     Usuario interage com app mobile (Cordova)
            ↓
    AngularJS captura a ação
            ↓
    Controller processa a lógica
            ↓
    View atualiza automaticamente (sem recarregar)
            ↓
    Cordova acessa recursos do celular (câmera, GPS, etc)

    Exmplo real:
    - Usuário clica em "tirar foto" -> Cordova acessa câmera
    - Foto é capturada -> AngularJS atualiza a tela dinamicamente
    - Tudo sem recarregar a página
}

VANTAGENS E DESVANTAGENS {
    ✅ VANTAGENS

    VANTAGEM                        | POR QUÊ
    ────────────────────────────────────────────────────────────────────────
    Código limpo e organizado       | MVC separa responsabilidades
    Desenvolvimento rápido          | Less repetitive, mais productive
    Comunidade grande               | Muito material e suporte
    Funciona bem com Cordova        | Perfeito para apps móveis
    Validação built-in              | Valida formulários automaticamente
    Two-way binding                 | Sincronização automática HTML ↔ JavaScript


    ❌ DESVANTAGENS

    DESVANTAGEM                     | PROBLEMA
    ────────────────────────────────────────────────────────────────────────
    Curva de aprendizado            | Precisa aprender conceitos novos (Services, Directives, etc)
    Performance em apps grandes     | Pode ficar lento com muitos dados
    AngularJS é antigo              | Criado em 2009, comunidade migrou
    Sem suporte oficial             | Google parou de manter (use Angular 2+ se precisar suporte)
    Compatibilidade                 | Não funciona bem com frameworks modernos
}

    QUANDO USAR AngularJS                      | QUANDO NÃO USAR
    ────────────────────────────────────────────────────────────────────────
    ✔️ Apps móveis com Cordova                 | ✖️ Apps muito grandes e complexas
    ✔️ Prototipagem rápida                     | ✖️ Projetos que precisam suporte oficial long-term
