Estrutura básica do AngularJS

Normalmente são organizados no projeto da seguinte forma:

index.js (ou app.js) -> configuração geral
controller.js -> lógica de cada tela
services.js -> comunicação / regras reutilizáveis

INDEX.JS {
    cria o módulo principal
    define dependências

    exemplo simples
    var app = angular.module("meuApp", []);

    app.run(function() {
        console.log("app iniciado");
    });

    o index.js é onde o app nasce. sem ele, nada funciona
}

CONTROLLER.JS {
    controla o comportamento da tela
    responde ações do usuário
    manipula dados pra exibir

    app.controller("MainCtrl", function($scope) {
        $scope.nome = "";

        $scope.salvar = function() {
            console.log("salvou " + $scope.nome);
        };
    });

    controller é quem manda na tela
    botão clicou → controller responde
    input mudou → controller atualiza

    controller NÃO deve:

    fazer requisição direto
    ter regra de negócio pesada
    ficar gigante
}

SERVICES.JS {
    centraliza regras
    faz requisições (API / SOAP)
    guarda lógica reutilizável

    app.service("UsuarioService", function($http) {
        this.buscarUsuarios = function() {
            return $http.get("/api/usuarios");
        };
    });
    
    service é onde fica a lógica que pode ser usada em vários lugares
}

FLUXO: HTML → Controller → Service → Controller → HTML

controller chamando service:
app.controller("MainCtrl", function($scope, UsuarioService) {
    $scope.usuarios = [];

    $scope.carregar = function() {
        UsuarioService.buscarUsuarios().then(function(response) {
            $scope.usuarios = response.data;
        });
    };
});

COMO A ARQUITETURA/MODULARIZAÇÃO DO ANGULARJS FUNCIONA: 

=======================================================
{
    o AngularJS tem um conceito central:
    módulo (module) {
        angular.module("meuApp", []);
        cria um módulo chamado "meuApp"
        nos colchetes [] ficam dependências (outros módulos) -> eles podem ser [internos]1 ou então [externos]2 {
            [1] ex: app.controllers
                    app.routes
                    comunica
                    services
                    sqlite

            [2] (bibliotecas) ex: ui.router
                                  ui.utils.masks
                                  ui.mask

                exemplo do ui.router

                ele registra coisas como:

                $stateProvider
                $state

                e você usa assim:

                app.config(function($stateProvider) {
                    $stateProvider.state(...);
                });
        }

        Quando fazemos por exemplo app.controller("MainCtrl", function() {});
        o angular faz algo como "ok, quando alguém pedir MainCtrl, eu sei como criar"
        isso vale pra tudo
        controller → Angular sabe instanciar
        service → Angular sabe injetar
        directive → Angular sabe aplicar no HTML

        isso se chama

        Dependency Injection (injeção de dependência)

        exemplo real
        app.controller("MainCtrl", function($scope, UsuarioService) {});

        Angular faz:

        1. vê que precisa de UsuarioService
        2. procura no módulo
        3. encontra
        4. injeta automaticamente

        ponto MAIS importante

        o Angular NÃO executa tudo na hora

        ele só:

        registra
        guarda
        usa quando necessário

        criar módulo
        angular.module("app", []);

        cria

        acessar módulo existente
        angular.module("app");

        pega o já criado
    }
    
    dentro desse módulo você encontra tipos de registro/artefatos do módulo diferentes:
    
    -----------------------------
    1. lógica / dados 
    service
    factory

    função: lógica reutilizável / integração / regras

    {
        factory 

        você retorna um objeto manualmente:

        app.factory('ComunicaService', function() {
            var service = {};

            service.buscar = function() {
            };

            return service;
        });

        ________________________________

        service

        o Angular instancia com new e você usa this:

        app.service('ComunicaService', function() {
            this.buscar = function() {
            };
        });

        RESUMO:
        os dois na prática funcionam da mesma forma; service é mais "orientado
        a classe", factory é mais flexível e podemos retornar qualquer coisa.
    }

    -----------------------------

    2. controle de tela
    controller

    função: ligar HTML com lógica

    -----------------------------

    3. interface (view layer)
    directive
    component

    função: mexer/criar elementos na tela
    
    {
        DIRECTIVE {
            é uma forma de criar comportamento ou estrutura no HTML

            serve para:

            reutilizar código de interface
            manipular DOM
            criar atributos/elementos personalizados
            exemplo simples:
            app.directive("meuBotao", function() {
                return {
                    template: '<button>Clique aqui</button>'
                };
            });

            uso:

            <meu-botao></meu-botao>
            exemplo com comportamento:
            app.directive("alertaClick", function() {
                return {
                    link: function(scope, element) {
                        element.on("click", function() {
                            alert("clicou");
                        });
                    }
                };
            });

            explicação:
            template → HTML que será renderizado
            link → onde você controla comportamento

            quando usar directive?
            quando precisa manipular DOM
            quando precisa criar comportamento reutilizável
            quando precisa criar algo tipo ng-click, ng-model
        }

        COMPONENT {
            é uma forma mais simples e organizada de criar interface

            basicamente uma directive simplificada

            exemplo:
            app.component("usuarioCard", {
                template: '<div>{{ $ctrl.nome }}</div>',
                
                controller: function() {
                    this.nome = "Nícolas";
                }
            });

            uso:

            <usuario-card></usuario-card>
            explicação:
            $ctrl → referência do controller do componente
            template + lógica no mesmo lugar
            mais organizado que directive

            quando usar component?
            criar partes reutilizáveis da tela
            montar UI modular
            substituir directives complexas

            diferença prática:
            directive → baixo nível (mais poder, mais complexidade)
            component → alto nível (mais simples, mais organizado)
        }
    }
    
    -----------------------------

    4. configurações do módulo
    config 

    bloco de configuração do módulo

    função: configurar coisas do framework {
        config não é bem um artefato do tipo controller/service/directive.

        ele é mais um bloco de configuração do módulo

        exemplo:

        angular.module('app').config(['$stateProvider', function($stateProvider) {
            // configura rotas aqui
        }]);

        o que ele faz

        o config roda na fase de configuração do AngularJS, antes da aplicação começar de verdade.

        ele é usado para configurar coisas do framework, por exemplo:

        rotas
        providers
        interceptors
        defaults de bibliotecas
    }

    5. filtro
    filter

    função: transforma valores para exibição

    app.filter('maiusculo', function() {
        return function(valor) {
            return String(valor).toUpperCase();
        };
    });

    uso:

    {{ nome | maiusculo }}

    > - artefatos menos utilizados - < 
    6. constant {
        valor fixo disponível em toda a app

        app.constant('API_URL', 'https://api.exemplo.com');
    }
    
    7. value {
        parecido com constant, mas menos “fixo” em termos de ciclo de config

        app.value('versao', '1.0');
    }

    8. provider {
        forma mais avançada de criar serviços configuráveis;
        permite configurar no .config

        app.provider('configuracao', function() {
            this.opcao = true;

            this.$get = function() {
                return {
                    opcao: this.opcao
                };
            };
        });
    }

    > - boa prática - <
    Utilizar array annotation na injeção de dependências:

    app.controller('MainCtrl', ['$scope', '$http', function($scope, $http) {}]);

    ao invés de

    app.controller('MainCtrl', function($scope, $http, $scope, $http) {});

    --------
    $inject

    melhor quando você quer organização e funções nomeadas

    function Funcoes($http) {
        return {};
    }

    Funcoes.$inject = ['$http'];

    app.factory('Funcoes', Funcoes);
}
